# Code of Conduct

## Purpose

This project aims to foster an open, respectful, and technically rigorous environment for discussing and developing the DIVE protocol.

Participation in this repository implies adherence to the following principles.

## Expected Behavior

Participants are expected to:

- Be respectful and professional in all interactions
- Focus on technical merit and constructive discussion
- Provide clear, evidence-based arguments
- Accept feedback and differing viewpoints
- Keep discussions relevant to the project

## Unacceptable Behavior

The following behaviors are not tolerated:

- Personal attacks or harassment
- Discriminatory or offensive language
- Trolling or intentionally disruptive behavior
- Dismissing contributions without technical justification
- Any conduct that would be inappropriate in a professional standards body

## IETF Alignment

This project follows the spirit of the IETF community guidelines:

- Open participation
- Rough consensus and running code
- Respectful technical debate

Contributors are encouraged to act consistently with IETF best practices when discussing protocol design.

## Handling Issues

If unacceptable behavior occurs:

- It may be addressed through warnings or moderation
- Comments or contributions may be removed
- Persistent issues may result in exclusion from the project

## Reporting

If you experience or witness problematic behavior, you may:

- Open an issue (for non-sensitive cases)
- Contact the maintainer directly:

[mateo@callec.net](mailto:mateo@callec.net)

## Scope

This Code of Conduct applies to:

- GitHub issues
- Pull requests
- Discussions related to this repository

---

The goal is to maintain a productive environment focused on building a robust and secure protocol specification.
